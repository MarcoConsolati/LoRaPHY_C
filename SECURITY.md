# Security Policy

## Supported Versions

Currently, the following versions of LoRaPHY_C are supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do NOT open public issues for security vulnerabilities.**

To report a security vulnerability, please email me directly at: **marco.consolati87@gmail.com**

### What to expect:
- **Response within 48 hours** - I'll acknowledge your report
- **Investigation** - I'll investigate and confirm the vulnerability
- **Fix development** - For confirmed issues, I'll work on a patch
- **Disclosure** - After fixing, I'll credit you (if desired) and publish the fix

### Scope:
This security policy applies to the core LoRaPHY_C implementation, including:
- LoRa modulation/demodulation components
- FFTW integration
- CRC calculation modules
- Memory management functions

Thank you for helping keep LoRaPHY_C and its users safe!
